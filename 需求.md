# Planner 需求文档

## 背景

用于记录每日计划和总结，提供方便的手机端操作体验。TODO 数据存储在 GitHub 仓库，重要日程同步到 Google Calendar 获得原生通知。

---

## 整体布局

三个主 Tab，左右滑动切换：

```
┌─────────────────────────────┐
│   TODO   │   习惯   │   日程  │
└─────────────────────────────┘
```

---

## Tab 1：TODO 清单

### 布局

```
┌──────────────────────┐
│ 全部  一场  生活  +   │  ← 分类标签（横向滚动，可新增）
│──────────────────────│
│ □ [高] 预约7月场地    │
│      已安排：5/23 19:00│
│ □ 写文稿              │
│ □ 洗鞋               │
│ ─────── 已完成 ─────  │
│ ✓ 给大拿回消息        │
│                  ＋  │
└──────────────────────┘
```

### 功能

- 顶部分类标签横向滚动，点 **＋** 可新增自定义分类（如：想读的书、想看的电影等）
- 默认分类：全部、一场、生活、脱口秀、工作、家务
- 每条任务显示优先级标签（高 / 普通）
- 已安排进日程的任务显示提示："已安排：5/23 19:00"
- 已完成的任务收起在底部"已完成"区域，可展开查看
- 点 **＋** 新增任务，填写：内容、分类、优先级

### 每张习惯卡包含

- 习惯名称、所属分类
- 目标频率：每天 / 每周 N 次
- 目标时长（可选，如每次 20 分钟）
- 本周七天完成情况（● 已完成，○ 未完成）
- 本周进度：已完成次数 / 目标次数
- 点"今天记录"打卡，可填写当次时长

### 记录弹出表单

```
┌─────────────────────────────┐
│  记录：日语口语               │
│                             │
│  日期    2026-05-23（今天）  │
│  时长    [20] 分钟           │
│  备注    ____________        │
│                             │
│         取消        保存    │
└─────────────────────────────┘
```

### 新增习惯

点右下角 ＋ 弹出表单，填写：
- 习惯名称
- 所属分类
- 目标频率（每天 / 每周 N 次）
- 目标时长（可选）

### 与日程联动

- 打卡记录完成后，可选择在今天日程里自动添加一条记录（如"日语口语 20min"）
- 习惯完成记录也会纳入**每周回顾**的统计

## Tab 3：日程

### 三种视图，顶部 Tab 切换

#### 日视图（默认）

```
┌──────────────────────┐
│  日    周    月        │
│──────────────────────│
│   ◀  2026-05-23  ▶   │
│                      │
│  09:00               │
│  10:00               │
│  ...                 │
│  19:00  📅 预约7月场地 │  ← 📅 表示已同步到 Google 日历
│  21:00  洗澡          │
│                  ＋  │
└──────────────────────┘
```

- 左右滑动或点箭头切换日期
- 已同步到 Google Calendar 的事件显示 📅 图标
- 点 **＋** 弹出新增表单

#### 周视图
- 横向显示 7 天
- 每天用色块标注已有安排，不同分类用不同颜色
- 点击某天跳转日视图

#### 月视图
- 标准日历格式
- 有安排的日期显示小圆点
- 点击某天跳转日视图

### 新增日程（弹出表单）

```
┌──────────────────────┐
│  新增日程             │
│                      │
│  日期    2026-05-23  │
│  开始    19:00        │
│  结束    21:00        │
│  提醒    [15分钟前 ▼] │  ← 仅同步到 Google 日历时生效
│  关联TODO  [下拉选择 ▼]│
│  或自填   __________  │
│                      │
│  ☐ 同步到 Google 日历 │  ← 勾选后才创建 Google 日历事件
│                      │
│      取消      保存   │
└──────────────────────┘
```

- 不勾选：仅存本地 JSON
- 勾选：调用 Google Calendar API 创建事件，并可设置提醒时间
- 删除时：有 Google 事件的自动同步删除

---

## TODO 与日程联动

- 新增日程时可从下拉列表关联一个 TODO 条目
- 在日视图勾选时间块完成 → 对应 TODO 自动标记完成
- 在 TODO 中勾选完成 → 对应日程时间块自动标记完成
- 已关联的 TODO 在清单中显示"已安排：5/23 19:00"

---

## Google Calendar 集成

- 首次使用时用 Google 账号授权（OAuth 2.0），授权一次后长期有效
- 新增日程时若勾选"同步"，调用 Google Calendar API 创建事件
- 删除日程时自动同步删除对应的 Google Calendar 事件
- Google 事件 ID 存在 `data.json` 中用于后续更新和删除
- 同步到 Google Calendar 的事件可按分类设置颜色（与 APP 内颜色一致）

---

## 技术方案

- 纯 HTML / CSS / JavaScript，无框架依赖
- 部署在 GitHub Pages，手机浏览器打开即用，可添加到主屏幕（PWA）
- GitHub REST API：读写 TODO 和 data.json
- Google Calendar API：创建、更新、删除日程事件
- GitHub Token 和 Google OAuth Token 均存在浏览器 localStorage

## 建议补充的功能（供讨论）

### 1. 分类颜色
每个分类设一个颜色，在日程时间轴、周视图色块、以及同步到 Google Calendar 的事件上统一使用，一眼就能区分是哪类事情。

### 2. 提醒时间可选
新增 Google Calendar 事件时可选提醒时间（15 分钟前 / 30 分钟前 / 1 小时前等），不用每次进 Google Calendar 手动改。

### 3. 快速新增入口
主页面右下角始终显示一个悬浮的 **＋** 按钮，点了弹出选择：是新增 TODO 还是新增日程，减少操作步骤。

### 4. 离线支持
手机没网的时候仍然可以新增和编辑，等有网了自动同步到 GitHub 和 Google Calendar。对手机端体验影响很大。

### 5. 每周回顾
每周日可以看一个简单的周总结：本周完成了多少 TODO、安排了多少日程。跟你现在手写"计划和总结"的习惯对应起来。

## 开发顺序建议

1. **基础框架**：三个 Tab、TODO 增删改查、数据存 GitHub
2. **习惯打卡**：习惯卡片、每周七天追踪、打卡记录
3. **日程功能**：日视图、新增表单、与 TODO 联动
4. **Google Calendar 同步**：OAuth 授权、事件创建和删除
5. **周视图 / 月视图**：导航用
6. **优化细节**：颜色、离线支持、快速新增、每周回顾
